# Formal Verification: Lean4 Proofs

Auto-generated Lean4 translation of `gf2-core` field arithmetic via
[Aeneas](https://github.com/AeneasVerif/aeneas)/[Charon](https://github.com/AeneasVerif/charon).

## What's here

- `Gf2Core/Types.lean` — Type definitions (`Fp<P>`, `MontConsts<P>`, trait structs)
- `Gf2Core/Funs.lean` — Function bodies (Montgomery arithmetic, field operations)
- `Gf2Core/FunsExternal.lean` — Axioms for external functions (std library ops)
- `Gf2Core/FunsExternal_Template.lean` — Template generated by Aeneas

Currently extracts: **`gfp/`** (prime field `Fp<const P: u64>` with Montgomery
multiplication) and **`gfpn/`** (tower extensions `QuadraticExt`, `CubicExt`
over the `ExtConfig` trait). See [WORKAROUNDS.md](WORKAROUNDS.md) for
post-processing workarounds and what's excluded.

## Prerequisites

1. **Lean4** via [elan](https://github.com/leanprover/elan):
   ```
   curl -sSf https://raw.githubusercontent.com/leanprover/elan/master/elan-init.sh | sh
   ```

2. **Charon** (local patched build, requires `nightly-2026-02-07`):
   ```
   rustup toolchain install nightly-2026-02-07
   rustup component add rustc-dev llvm-tools-preview rust-src --toolchain nightly-2026-02-07
   ```
   Our pipeline uses a patched Charon built from `/data/aeneas-build/charon/`
   (upstream base `24a17b5e` + 3 local HRTB/associated-type fixes).
   See `dev/lean4-verification-pipeline.md` for full details and rebuild
   instructions.

3. **Aeneas** (requires OCaml 5 via opam):
   ```
   opam init && eval $(opam env)
   opam install ppx_deriving visitors easy_logging zarith yojson core_unix \
     odoc ocamlgraph menhir unionFind zarith progress domainslib dune
   git clone --depth 1 https://github.com/AeneasVerif/aeneas.git /tmp/aeneas
   cd /tmp/aeneas && make setup-charon && make build-dev
   cp /tmp/aeneas/bin/aeneas ~/.cargo/bin/
   ```

## Running the pipeline

```bash
./scripts/verify-lean.sh
```

This runs: Charon extraction → Aeneas translation → post-processing fixes →
`lake build`. All generated files are committed so you can also just build:

```bash
cd proofs && lake build
```

## Regenerating

If Rust source in `gfp/` or `gfpn/` changes, regenerate and commit:

```bash
./scripts/verify-lean.sh
git add proofs/Gf2Core/
```
